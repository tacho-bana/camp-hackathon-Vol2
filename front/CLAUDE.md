## プロジェクト概要

- プロジェクト名: 未定
- 一言説明: 現実世界の外出を防衛拠点の設置行動に変換する、位置情報連動型のタワーディフェンスゲーム。家のルーターにウイルスが侵入しようとしてくるのを、現実の街で調達した防衛拠点で守る。
- 対象ユーザー: 未定

## 技術スタック

- フレームワーク: React (Vite)
- 言語: TypeScript
- UIライブラリ: 未定
- CSSフレームワーク: 未定
- 地図ライブラリ: Mapbox GL JS
- バックエンド: FastAPI (Python) ※別チーム担当、フロント側は触らない
- データベース: PostgreSQL (Supabase)
- 認証: 未定（MVPでは一旦無視）
- デプロイ先: フロント = Vercel / バックエンド = Render
- パッケージマネージャー: 未定

## ページ構成

- /login : ログイン画面（新規登録・ログイン）※MVPでは一旦無視
- /map : 地図画面（ゲーム開始前・準備フェーズ・バトル中を兼ねる）
- /result : 結果画面（勝ち負けのみ表示）
- /settings : 設定画面（ログアウト、ユーザー情報）※優先度低

## 機能一覧

優先度順（1セッション完結型 / A案）:

1. 地図表示（Mapbox GL JS）
2. 現在地取得・表示
3. 開発用の位置偽装モード（地図タップで現在地を動かせる）※開発効率のため必須
4. ゲーム開始ボタン → 現在地が「家（本拠地）」になる
5. 周辺の店舗（POI）を Mapbox のデータから表示
6. 準備フェーズ（制限時間 15分）
7. 店から半径 50m 以内で「チェックイン」ボタンが出現
   - チェックインで +30 ビットコイン回復
   - 1店1回のみ、クールダウンなし
8. 店から半径 50m 以内で「防衛拠点設置」ができる
   - コスト: 150 ビットコイン
   - 1店につき1拠点まで
9. ゲームスタートボタン → 敵Wave 開始（5分）
10. 敵スポーン（家から半径 100m の外周からランダム、合計 50体）
11. 敵が家に向かって進行
    - 速い敵（HP低め・速い）
    - 遅い敵（HP高め・遅い）
12. 敵は「家」と「拠点」のうち近い方を攻撃
13. 拠点が自動で敵を迎撃
    - 攻撃範囲: 半径 30m
    - 攻撃間隔: 1秒に1回
    - ダメージ: 敵を3発で倒せる
    - 拠点HP: 100
14. 敵の拠点・家への攻撃: 5ダメージ
15. 家HP: 100、0になったら即敗北
16. 結果画面（勝ち/負けのみ）

## 初期値・ゲームパラメータまとめ

- 初期ビットコイン: 300
- 拠点設置コスト: 150 ビットコイン
- チェックイン回復量: 30 ビットコイン
- 準備フェーズ時間: 15分
- バトルフェーズ時間: 5分
- 敵の総数: 50体
- 敵のスポーン半径: 家から 100m
- 拠点の攻撃範囲: 30m
- 拠点のHP: 100
- 家のHP: 100
- 店の判定距離: 50m

## 特記事項

- 開発期間: 1週間（ハッカソン、オンライン提出型）
- チーム構成: 3人
  - フロント（React）: キムラさん、自分
  - バックエンド（FastAPI）: 遊佐さん、たちょさん
- **自分の開発範囲はフロントのみ。`back/` ディレクトリは読むのはOKだが、絶対に編集しない。**
- フロントの大枠（画面遷移）は GitHub Copilot で既に生成済み。
- MVPでは以下を切る:
  - 認証機能
  - 店固有の能力（全拠点を同じ能力に統一）
  - ボス
  - 音・演出
  - PvP・協力プレイ
- サーバー権威型の設計（ルール判定はすべてバックエンド側）。
- ユーザーは実際に街を歩いて拠点設置する前提だが、開発中は位置偽装モードでテストする。
- デモは事前に実地で録画した動画で行う。
- 絵はキムラさんが担当。
- 世界観: ルーターに侵入しようとするコンピュータウイルスが攻めてくる設定。
